Design notes — conversation view (dark)

  • Tokenized palette (§11.4). All colors now come from CSS custom properties keyed off data-theme × data-theme-mode. Default is Hearth, dark variant; flip the switcher in the top-right to try Linen / Vellum / Twilight, or toggle light mode in place.
  • Warm dark, not cold. In the Hearth default, base is #1E1410 (slight brown-black), bubbles #33221A, ink #F2E2CB cream. Vellum dark swaps in a true slate; Twilight dark is aubergine. No pure neutrals in any palette.
  • Familiar accent lifts in dark mode. Ollie's accent comes from --accent-familiar, which is intentionally brighter per-theme in the dark variant (Hearth ochre → #E0B25A, Linen forest → #7CB5A8, Vellum gold → #D4AE5C, Twilight sage → #9ECDA8) so the familiar still reads "alive" at low brightness.
  • User bubble stays "you". Deep tint of --accent-user backs the message; this is the place to push back if it feels too "Valentines" in any given palette.
  • Bot bubble keeps its left-edge rule. The --rule-strong token is bound to the familiar accent in every theme, so the visual language (hex avatar + AI pill + left-edge rule = familiar) carries through all eight palette × mode combinations.
  • Glow on the attachment, not the chrome. The only soft luminance in the view is the sun in the attachment photo — chrome stays matte. Keeps the "magical undertone" in the content, not the UI shell.
