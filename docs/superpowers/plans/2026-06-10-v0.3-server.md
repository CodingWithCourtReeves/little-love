# LittleLove v0.3 — Server Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the LittleLove server from the v0.2 "exactly two humans per room, one body per message" model to v0.3's "one human partner + N familiars per room, fan-out bodies addressed per recipient" model. Add the bot-registration REST surface, the room-management WS frames (`CreateRoom`, `RenameRoom`, `LeaveRoom` and their server-emitted twins), the multi-member `Rooms`/`RoomCreated` payload, and the per-recipient `messages` schema. Migration 0006 is stop-the-world (acceptable for 2 users; see spec §10.1).

**Architecture:** The wire shape and the schema both pivot to "per-recipient." `accounts` grows `is_bot`, `owner_account_id`, `partner_account_id`; `rooms` grows `name`; `messages` grows `recipient_account_id` (PK becomes composite). The v0.2 `room_members_one_per_account` unique index is dropped — monogamy moves into app-layer checks against `accounts.partner_account_id` at `ConsumeInvite` and `CreateRoom`. All Ed25519 signatures keep the §8.5.1 domain-separation pattern; two new tags (`littlelove.v0.3.bot-register`, `littlelove.v0.3.bot-delete`) verify owner consent for bot lifecycle. Server stays content-blind: every per-recipient body is opaque ciphertext.

**Tech Stack:** Rust 1.88, Axum 0.7, sqlx 0.7 + Postgres, ed25519-dalek 2, base64 0.22, ulid 1, sha2 0.10, `littlelove-crypto` workspace crate.

**Out of scope:** Client UI (separate plan `2026-06-10-v0.3-client.md`). Bot binary (separate plan `2026-06-10-v0.3-bot.md`). v0.2 bot reseed to owner-derived identity (spec §10.3 keeps existing bots as-is). Session-locking for two bot processes (spec §4.3, deferred). Member addition to existing rooms (spec §2 non-goal). Read receipts, message edit/delete.

**Spec sections implemented:** §2.x (goals/non-goals affirmed), §3.x (data model), §4.x (bot lifecycle REST), §5.x (room creation flow + monogamy), §6.x (per-conversation encryption — fan-out validation only; the crypto stays on the client), §7.x (room name + sidebar + leave/cascade), §8.x (REST + WS contracts), §9 (schema migration), §10.1 (server migration mechanics), §10.3 (v0.2 bot migration helper), §11 acceptance criteria 1–7 + 10.

**Spec ambiguity flagged and resolved by this plan:**
- §5.2 step 5 says the consumer learns about the new room via `RoomCreated` (broadcast to all members including the consumer). §8.2 lists `InviteConsumed` under "Existing frames unchanged: semantics preserved." This plan **keeps** `InviteConsumed` for the consumer (single-shot, server → consumer only) and emits `RoomCreated` to every other member. Both frames carry the new v0.3 `members` array shape. This preserves the v0.2 client's "consumer waits for InviteConsumed" code path while making RoomCreated the broadcast frame the rest of the room hears. If Court prefers dropping `InviteConsumed` entirely, amend the spec before Task 21 starts.

## Conventions & Test Infrastructure

- **DB-touching tests** use the existing `server/tests/common/mod.rs` helpers (`setup_db`, `reset_tables`). For v0.3, extend `reset_tables` to truncate using `RESTART IDENTITY CASCADE` so the new FK columns reset cleanly.
- **Test isolation:** `serial_test::serial` on tests that mutate `accounts`/`rooms`/`messages` to avoid races.
- **Wire format:** every frame keeps the `kind` discriminator (spec §8.2). New frames join `RoomClientFrame` / `RoomServerFrame` enums.
- **Domain separation:** every Ed25519 sig is `tag || 0x00 || payload`. Two new tags this round; both verified with `verify_strict` per the v0.2 §8.5.1 rationale.
- **No backwards-compat shim.** The v0.2 client cannot talk to a v0.3 server (Send.body shape is gone). Spec §10.2 acknowledges this; both clients are upgraded before server deploy.
- **Per-recipient `messages.body`:** still XChaCha20-Poly1305 wire envelope; the server is content-blind. The server only knows the row exists for a given `(message_id, recipient_account_id)`.
- **Commit cadence:** one commit per task. Required PR checks before merge: `cargo fmt --check`, `cargo clippy -- -D warnings`, `cargo test`. Standing project rule.

```sql
-- 0006_v0_3_partner_and_bot.sql

-- accounts: bot flag + ownership + canonical partner link.
ALTER TABLE accounts
  ADD COLUMN is_bot              BOOLEAN NOT NULL DEFAULT FALSE,
  ADD COLUMN owner_account_id    BIGINT  REFERENCES accounts(id) ON DELETE CASCADE,
  ADD COLUMN partner_account_id  BIGINT  REFERENCES accounts(id) ON DELETE SET NULL;

ALTER TABLE accounts
  ADD CONSTRAINT accounts_partner_not_self
    CHECK (partner_account_id IS NULL OR partner_account_id <> id);

ALTER TABLE accounts
  ADD CONSTRAINT accounts_owner_only_for_bots
    CHECK ((is_bot AND owner_account_id IS NOT NULL)
        OR (NOT is_bot AND owner_account_id IS NULL));

ALTER TABLE accounts
  ADD CONSTRAINT accounts_bots_no_partner
    CHECK (NOT is_bot OR partner_account_id IS NULL);

CREATE INDEX accounts_owner_idx   ON accounts(owner_account_id);
CREATE INDEX accounts_partner_idx ON accounts(partner_account_id);

-- rooms: optional display name (empty = client derives from member roles).
ALTER TABLE rooms
  ADD COLUMN name TEXT NOT NULL DEFAULT '',
  ADD CONSTRAINT rooms_name_length CHECK (char_length(name) <= 64);

-- room_members: drop v0.2 monogamy index; partner check moves to app layer.
DROP INDEX room_members_one_per_account;
```

```sql
-- messages: switch from one-row-per-message to one-row-per-recipient.
-- The recipient column is initially nullable so we can backfill before
-- enforcing NOT NULL.
ALTER TABLE messages
  ADD COLUMN recipient_account_id BIGINT REFERENCES accounts(id) ON DELETE CASCADE;

-- Backfill: for each existing v0.2 row, the recipient is the room member
-- who isn't the sender. v0.2's `room_members_one_per_account` unique index
-- guaranteed exactly one other member per room, so LIMIT 1 is unambiguous.
UPDATE messages m
SET recipient_account_id = (
  SELECT rm.account_id
  FROM room_members rm
  WHERE rm.room_id = m.room_id
    AND rm.account_id <> m.from_account_id
  LIMIT 1
)
WHERE recipient_account_id IS NULL;

ALTER TABLE messages
  ALTER COLUMN recipient_account_id SET NOT NULL;

-- v0.2 had PK on `messages.id` (text/ULID). In v0.3 the same logical
-- message has N rows (one per recipient), so the PK becomes composite.
ALTER TABLE messages
  DROP CONSTRAINT messages_pkey,
  ADD PRIMARY KEY (id, recipient_account_id);

CREATE INDEX messages_room_recipient_idx
  ON messages(room_id, recipient_account_id, id);

-- Backfill partner_account_id from existing 2-human couple rooms.
UPDATE accounts a
SET partner_account_id = (
  SELECT b.id
  FROM room_members rm_a
  JOIN room_members rm_b ON rm_b.room_id = rm_a.room_id AND rm_b.account_id <> rm_a.account_id
  JOIN accounts b        ON b.id = rm_b.account_id
  WHERE rm_a.account_id = a.id
    AND b.is_bot = FALSE
  LIMIT 1
)
WHERE a.is_bot = FALSE AND a.partner_account_id IS NULL;
```

```rust
//! Verify migration 0006 backfills recipient_account_id and partner_account_id
//! from a seeded v0.2-shaped dataset.

mod common;

#[tokio::test]
#[serial_test::serial]
async fn backfill_populates_recipient_and_partner() {
    // setup_db runs all migrations including 0006. We then seed a synthetic
    // v0.2-style record set by INSERTing accounts + a 2-member room + a
    // message, deliberately bypassing migration logic, then assert the
    // backfill state (which has already run as part of `migrate`).
    //
    // Because migrate runs once at connect, this test inserts AFTER backfill;
    // for the actual backfill to fire, we use the new explicit-call helper
    // `common::apply_v0_3_backfills(pool)` that re-runs the UPDATE statements
    // idempotently against the seeded rows.

    let store = common::setup_db().await;
    let pool = store.pool();
    common::reset_tables(pool).await;

    // 1. Seed two humans as if they were v0.2 accounts.
    let court_id: (i64,) = sqlx::query_as(
        "INSERT INTO accounts (username, ed25519_pub, x25519_pub)
         VALUES ('court', $1, $2) RETURNING id")
        .bind(vec![0u8; 32]).bind(vec![1u8; 32])
        .fetch_one(pool).await.unwrap();
    let kaitlyn_id: (i64,) = sqlx::query_as(
        "INSERT INTO accounts (username, ed25519_pub, x25519_pub)
         VALUES ('kaitlyn', $1, $2) RETURNING id")
        .bind(vec![2u8; 32]).bind(vec![3u8; 32])
        .fetch_one(pool).await.unwrap();

    // 2. Seed a v0.2-style 2-member room.
    sqlx::query("INSERT INTO rooms (id) VALUES ('01J_legacy')")
        .execute(pool).await.unwrap();
    for who in [court_id.0, kaitlyn_id.0] {
        sqlx::query("INSERT INTO room_members (room_id, account_id) VALUES ('01J_legacy', $1)")
            .bind(who).execute(pool).await.unwrap();
    }

    // 3. Insert a v0.2-style message (recipient_account_id starts NULL).
    sqlx::query(
        "INSERT INTO messages (id, room_id, from_account_id, body, recipient_account_id)
         VALUES ('msg_a', '01J_legacy', $1, 'CT', NULL)")
        .bind(court_id.0).execute(pool).await.unwrap();

    // 4. Apply the backfill statements explicitly.
    common::apply_v0_3_backfills(pool).await;

    // 5. Assert recipient is now Kaitlyn.
    let (rcpt,): (i64,) = sqlx::query_as(
        "SELECT recipient_account_id FROM messages WHERE id='msg_a'")
        .fetch_one(pool).await.unwrap();
    assert_eq!(rcpt, kaitlyn_id.0);

    // 6. Assert partner_account_id is set on both.
    let (court_p,): (Option<i64>,) = sqlx::query_as(
        "SELECT partner_account_id FROM accounts WHERE id=$1")
        .bind(court_id.0).fetch_one(pool).await.unwrap();
    let (kait_p,):  (Option<i64>,) = sqlx::query_as(
        "SELECT partner_account_id FROM accounts WHERE id=$1")
        .bind(kaitlyn_id.0).fetch_one(pool).await.unwrap();
    assert_eq!(court_p, Some(kaitlyn_id.0));
    assert_eq!(kait_p,  Some(court_id.0));
}
```

```rust
/// Domain-separation tag for `DELETE /accounts/bot/{label}`. Spec §4.x + §8.3.
/// Signed input: `BOT_DELETE_DOMAIN_TAG || 0x00 || label_utf8`
```

```rust
#[derive(Debug, thiserror::Error, PartialEq, Eq)]
pub enum MonogamyError {
    #[error("the other account is not your partner")]
    WrongPartner,
}

/// Validate that consumer `me` can pair with inviter `peer`.
/// - Both NULL → ok (set both on commit).
/// - me.partner == peer.id AND peer.partner == me.id → ok (already paired).
/// - any other combination → WrongPartner.
pub async fn monogamy_check(
    pool: &sqlx::PgPool,
    me: i64,
    peer: i64,
) -> sqlx::Result<Result<(), MonogamyError>> {
    let row: Option<(Option<i64>, Option<i64>)> = sqlx::query_as(
        "SELECT (SELECT partner_account_id FROM accounts WHERE id=$1),
                (SELECT partner_account_id FROM accounts WHERE id=$2)"
    ).bind(me).bind(peer).fetch_optional(pool).await?;
    let (mine, theirs) = row.unwrap_or((None, None));
    let ok = matches!((mine, theirs),
        (None, None) | (Some(p), Some(q)) if p == peer && q == me);
    Ok(if ok { Ok(()) } else { Err(MonogamyError::WrongPartner) })
}

/// Set `partner_account_id` on both accounts in a single transaction.
/// Idempotent on already-paired-with-each-other.
pub async fn set_partner_link(pool: &sqlx::PgPool, a: i64, b: i64) -> sqlx::Result<()> {
    let mut tx = pool.begin().await?;
    sqlx::query("UPDATE accounts SET partner_account_id=$2 WHERE id=$1 AND (partner_account_id IS NULL OR partner_account_id=$2)")
        .bind(a).bind(b).execute(&mut *tx).await?;
    sqlx::query("UPDATE accounts SET partner_account_id=$2 WHERE id=$1 AND (partner_account_id IS NULL OR partner_account_id=$2)")
        .bind(b).bind(a).execute(&mut *tx).await?;
    tx.commit().await?;
    Ok(())
}

/// True iff `bot_id` is a familiar owned by `human_id`, or by their partner.

pub async fn bot_owned_by(pool: &sqlx::PgPool, bot_id: i64, human_id: i64) -> sqlx::Result<bool> {

                 .unwrap_or(0::bigint) AS owner,
```

```rust
/// Create a room with `host` as the initiating human and `bots` as the
/// pre-joined familiar account_ids. The host's partner is NOT added; if
/// they're in the room it's via a subsequent ConsumeInvite. Returns the
/// ULID `room_id`. Caller validates bot ownership via `bot_owned_by` before
/// calling — this function does not re-check.
pub async fn create_room_with_members(
    pool: &sqlx::PgPool,
    host: i64,
    bots: &[i64],
    name: String,
) -> Result<String, CreateRoomError> {
    let room_id = ulid::Ulid::new().to_string();
    let mut tx = pool.begin().await?;
    sqlx::query("INSERT INTO rooms (id, name) VALUES ($1, $2)")
        .bind(&room_id).bind(&name).execute(&mut *tx).await?;
    let all_members: Vec<i64> = std::iter::once(host).chain(bots.iter().copied()).collect();
    for account_id in all_members {
        sqlx::query("INSERT INTO room_members (room_id, account_id) VALUES ($1, $2)
                     ON CONFLICT DO NOTHING")
            .bind(&room_id).bind(account_id).execute(&mut *tx).await?;
    }
    tx.commit().await?;
    Ok(room_id)
}

pub async fn leave_room(pool: &sqlx::PgPool, room_id: &str, account_id: i64) -> sqlx::Result<LeaveOutcome> {
    let mut tx = pool.begin().await?;
    sqlx::query("DELETE FROM room_members WHERE room_id=$1 AND account_id=$2")
        .bind(room_id).bind(account_id).execute(&mut *tx).await?;
    // Count remaining humans.
    let (humans,): (i64,) = sqlx::query_as(
        "SELECT COUNT(*) FROM room_members m
         JOIN accounts a ON a.id = m.account_id
         WHERE m.room_id = $1 AND a.is_bot = FALSE"
    ).bind(room_id).fetch_one(&mut *tx).await?;
    let room_deleted = if humans == 0 {
        sqlx::query("DELETE FROM rooms WHERE id=$1").bind(room_id).execute(&mut *tx).await?;
        true
    } else { false };
    tx.commit().await?;
    Ok(LeaveOutcome { room_deleted, remaining_member_usernames: () })
```

```rust
use chrono::{DateTime, Utc};
use sqlx::PgPool;

#[derive(Debug, Clone)]
pub struct MessageRow {
    pub id: String,                      // ULID, NOT unique on its own in v0.3
    pub room_id: String,
    pub from_account_id: i64,
    pub recipient_account_id: i64,
    pub body: String,                    // single addressed ciphertext
    pub ts: DateTime<Utc>,
}

#[derive(Clone)]
pub struct Store { pool: PgPool }

impl Store {
    pub async fn connect(database_url: &str) -> anyhow::Result<Self> {
        let pool = PgPool::connect(database_url).await?;
        sqlx::migrate!("./migrations").run(&pool).await?;
        Ok(Self { pool })
    }
    pub fn pool(&self) -> &PgPool { &self.pool }

    pub async fn insert(&self, row: MessageRow) -> anyhow::Result<()> {
        sqlx::query(
            "INSERT INTO messages (id, room_id, from_account_id, recipient_account_id, body, ts)
             VALUES ($1,$2,$3,$4,$5,$6)
             ON CONFLICT (id, recipient_account_id) DO NOTHING")
            .bind(&row.id).bind(&row.room_id).bind(row.from_account_id)
            .bind(row.recipient_account_id).bind(&row.body).bind(row.ts)
            .execute(&self.pool).await?;
        Ok(())
    }

    /// Replay every message in `room_id` addressed to `recipient`, after
    /// `since_message_id` (exclusive). Pass `None` for the whole room.
    pub async fn messages_for_recipient(
        &self,
        room_id: &str,
        recipient_account_id: i64,
        since_message_id: Option<&str>,
    ) -> anyhow::Result<Vec<MessageRow>> {
        let rows = if let Some(since) = since_message_id {
            sqlx::query_as::<_, (String,String,i64,i64,String,DateTime<Utc>)>(
                "SELECT id,room_id,from_account_id,recipient_account_id,body,ts
                 FROM messages
                 WHERE room_id=$1 AND recipient_account_id=$2 AND id > $3
                 ORDER BY id ASC")
                .bind(room_id).bind(recipient_account_id).bind(since)
                .fetch_all(&self.pool).await?
        } else {
            sqlx::query_as::<_, (String,String,i64,i64,String,DateTime<Utc>)>(
                "SELECT id,room_id,from_account_id,recipient_account_id,body,ts
                 FROM messages
                 WHERE room_id=$1 AND recipient_account_id=$2
                 ORDER BY id ASC")
                .bind(room_id).bind(recipient_account_id)
                .fetch_all(&self.pool).await?
        };
        Ok(rows.into_iter().map(|(id,room_id,from_account_id,recipient_account_id,body,ts)| MessageRow {
            id, room_id, from_account_id, recipient_account_id, body, ts,
        }).collect())
    }
}
```

```rust
async fn handle_create_room(
    state: &AppState, me: &AccountRecord,
    name: Option<String>, bot_ids: Vec<i64>, invite_partner: bool,
    tx: &mpsc::UnboundedSender<RoomServerFrame>,
) {
    let store = state.store.as_ref().expect("store");
    // Validate ownership for each bot.
    for bot_id in &bot_ids {
        match crate::rooms::bot_owned_by(store.pool(), *bot_id, me.id).await {
            Ok(true) => {}
            _ => { send_error(tx, error_codes::NOT_OWNED_BOT, ""); return; }
        }
    }
    let name = name.unwrap_or_default();
    if name.chars().count() > 64 {
        send_error(tx, "BadName", "name too long"); return;
    }
    let room_id = match crate::rooms::create_room_with_members(store.pool(), me.id, &bot_ids, name.clone()).await {
        Ok(id) => id,
        Err(crate::rooms::CreateRoomError::Db(e)) => {
            warn!("create_room: {e}"); send_error(tx, "Internal", ""); return;
        }
        Err(crate::rooms::CreateRoomError::AlreadyPaired(_)) => {
            // Can't happen in v0.3 — kept for the error variant. Treat as Internal.
            send_error(tx, "Internal", ""); return;
        }
    };

    // Optionally create a pending invite.
    let pending = if invite_partner {
        let (_, code, hash) = littlelove_crypto::invite::generate_invite();
        let expires_at = crate::invites::default_expiry(chrono::Utc::now());
        if let Err(e) = crate::invites::create_invite_record(store.pool(), me.id, &hash, expires_at).await {
            warn!("create_invite_record: {e}"); send_error(tx, "Internal", ""); return;
        }
        let qr = crate::invites::qr_png_base64(&code).unwrap_or_default();
        Some(crate::wire::PendingInvite { code, qr_png_base64: qr, expires_at })
    } else { None };

    // Push RoomCreated to host + bots' sessions.
    let detail = crate::rooms::room_detail(store.pool(), &room_id).await.unwrap().unwrap();

        members: detail.members.iter().cloned().map(crate::rooms::Member::into_wire).collect(),

    };
```

```rust
async fn handle_leave_room(
    state: &AppState, me: &AccountRecord, room_id: &str,
    tx: &mpsc::UnboundedSender<RoomServerFrame>,
) {
    let store = state.store.as_ref().expect("store");
    if !is_member(store.pool(), room_id, me.id).await.unwrap_or(false) {
        send_error(tx, error_codes::UNKNOWN_ROOM, ""); return;
    }
    // Capture remaining members BEFORE leave (so we know who to notify).
    let before = members_for_room(store.pool(), room_id).await.unwrap_or_default();
    let outcome = match crate::rooms::leave_room(store.pool(), room_id, me.id).await {
        Ok(o) => o,
        Err(e) => { warn!("leave_room: {e}"); send_error(tx, "Internal", ""); return; }
    };
    let frame = RoomServerFrame::MemberLeft { room_id: room_id.to_string(), username: me.username.clone() };
    for m in &before {
        if m.account_id == me.id { continue; }
        state.routing.deliver(&m.username, frame.clone()).await;
    }
    let _ = outcome; // room_deleted is informational; clients re-query Rooms on next connect.
}
```

```rust
#[derive(Debug, serde::Deserialize)]
pub struct CreateBotRequest {
    pub owner_username:  String,
    pub bot_label:       String,
    pub bot_username:    String,
    pub bot_ed25519_pub: String,
    pub bot_x25519_pub:  String,
    pub owner_signature: String,
}

#[derive(Debug, serde::Serialize)]
pub struct CreateBotResponse {
    pub account_id:   i64,
    pub bot_username: String,
    pub is_bot:       bool,
}

fn bot_label_ok(label: &str) -> bool {
    (1..=32).contains(&label.len())
        && label.bytes().all(|b| b.is_ascii_lowercase() || b.is_ascii_digit() || b == b'-')
}

pub async fn create_bot_account(
    State(state): State<AppState>,
    Json(req): Json<CreateBotRequest>,
) -> impl IntoResponse {
    let store = match state.store.as_ref() {
        Some(s) => s,
        None => return (StatusCode::INTERNAL_SERVER_ERROR, "store unavailable").into_response(),
    };
    if !bot_label_ok(&req.bot_label) {
        return (StatusCode::BAD_REQUEST, "invalid bot_label").into_response();
    }
    if !username_ok_v2(&req.bot_username) {
        return (StatusCode::BAD_REQUEST, "invalid bot_username").into_response();
    }
    let owner = match lookup_full_account(store, &req.owner_username).await {
        Ok(Some(a)) => a, _ => return (StatusCode::UNAUTHORIZED, "no such owner").into_response(),
    };
    let bot_ed = match decode_pubkey(&req.bot_ed25519_pub) {
        Some(b) => b, None => return (StatusCode::BAD_REQUEST, "bad bot_ed25519_pub").into_response(),
    };
    let bot_x = match decode_pubkey(&req.bot_x25519_pub) {
        Some(b) => b, None => return (StatusCode::BAD_REQUEST, "bad bot_x25519_pub").into_response(),
    };
    let sig = match B64.decode(&req.owner_signature) {
        Ok(s) => s, Err(_) => return (StatusCode::UNAUTHORIZED, "bad signature").into_response(),
    };
    if littlelove_crypto::sig::verify_bot_register_signature(&owner.ed25519_pub, &bot_ed, &sig).is_err() {
        return (StatusCode::UNAUTHORIZED, "signature did not verify").into_response();
    }
    // Idempotency: if a bot with the same (owner, label) already exists, return 200.
    let existing: Option<(i64, String)> = sqlx::query_as(
        "SELECT id, username FROM accounts WHERE owner_account_id=$1 AND is_bot=TRUE
         AND username=$2")
        .bind(owner.id).bind(&req.bot_username).fetch_optional(store.pool()).await.ok().flatten();
    if let Some((id, u)) = existing {
        return (StatusCode::OK, Json(CreateBotResponse { account_id: id, bot_username: u, is_bot: true })).into_response();
    }
    let row: Result<(i64,), sqlx::Error> = sqlx::query_as(
        "INSERT INTO accounts (username, ed25519_pub, x25519_pub, is_bot, owner_account_id)
         VALUES ($1,$2,$3,TRUE,$4) RETURNING id")
        .bind(&req.bot_username).bind(&bot_ed).bind(&bot_x).bind(owner.id)
        .fetch_one(store.pool()).await;
    match row {
        Ok((id,)) => (StatusCode::CREATED, Json(CreateBotResponse {
            account_id: id, bot_username: req.bot_username, is_bot: true,
        })).into_response(),
        Err(sqlx::Error::Database(db)) if db.code().as_deref() == Some("23505")
            => (StatusCode::CONFLICT, "bot_username taken").into_response(),
        Err(e) => { tracing::warn!("create_bot db: {e}"); (StatusCode::INTERNAL_SERVER_ERROR, "db error").into_response() }
    }
}

fn username_ok_v2(u: &str) -> bool {
    let len = u.len();
    (1..=32).contains(&len)
        && u.bytes().enumerate().all(|(i,b)|
            (i==0 && (b.is_ascii_lowercase() || b.is_ascii_digit()))
            || (i>0 && (b.is_ascii_lowercase() || b.is_ascii_digit() || b == b'_' || b == b'-')))
}
```

```rust
pub async fn delete_bot_account(

    Path(label): Path<String>,
    headers: HeaderMap,

        Some(s) => s, None => return (StatusCode::INTERNAL_SERVER_ERROR, "store unavailable").into_response(),
    };
    let owner_username = headers.get("X-Owner-Username").and_then(|v| v.to_str().ok()).map(|s| s.to_string());
    let sig_b64        = headers.get("X-Owner-Signature").and_then(|v| v.to_str().ok()).map(|s| s.to_string());
    let (Some(owner_u), Some(sig_b64)) = (owner_username, sig_b64) else {
        return (StatusCode::UNAUTHORIZED, "missing auth").into_response();
    };
    if !bot_label_ok(&label) {

    let owner = match lookup_full_account(store, &owner_u).await {

    };
    if littlelove_crypto::sig::verify_bot_delete_signature(&owner.ed25519_pub, label.as_bytes(), &sig).is_err() {
```

```rust
#[derive(Debug, Serialize)]
pub struct InvitePreviewResponse {
    pub room_id:    String,
    pub name:       String,
    pub members:    Vec<crate::wire::Member>,
    pub expires_at: DateTime<Utc>,
}

pub async fn preview_invite(
    State(state): State<AppState>,
    Path(code): Path<String>,
) -> impl IntoResponse {
    let store = match state.store.as_ref() {
        Some(s) => s, None => return (StatusCode::INTERNAL_SERVER_ERROR, "store unavailable").into_response(),
    };
    let canonical = match decode_code(&code) {
        Ok(t) => t, Err(_) => return (StatusCode::NOT_FOUND, "no such code").into_response(),
    };
    let token_hash = sha256(&canonical);
    let invite = match lookup_invite(store.pool(), &token_hash).await {
        Ok(Some(i)) => i,
        Ok(None) => return (StatusCode::NOT_FOUND, "no such code").into_response(),
        Err(e) => { tracing::warn!("lookup_invite: {e}"); return (StatusCode::INTERNAL_SERVER_ERROR, "db error").into_response(); }
    };
    match invite.state(Utc::now()) {
        InviteState::Expired => return (StatusCode::GONE, "expired").into_response(),
        InviteState::Consumed => return (StatusCode::GONE, "consumed").into_response(),
        InviteState::Pending => {}
    }
    let room_id = match room_for_invite(store.pool(), &token_hash).await {
        Ok(Some(r)) => r,
        _ => return (StatusCode::NOT_FOUND, "invite not bound to a room").into_response(),
    };
    let detail = match crate::rooms::room_detail(store.pool(), &room_id).await {
        Ok(Some(d)) => d,
        _ => return (StatusCode::NOT_FOUND, "room gone").into_response(),
    };
    Json(InvitePreviewResponse {
        room_id: detail.room_id,
        name: detail.name,
        members: detail.members.into_iter().map(crate::rooms::Member::into_wire).collect(),
        expires_at: invite.expires_at,
    }).into_response()
}
```
