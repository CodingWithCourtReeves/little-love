# LittleLove v0.3 — Client Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the Flutter client from the v0.2 "two-person room with peer fields" model to v0.3's "multi-member rooms with per-recipient fan-out and a new Create-Chat wizard." The wire-layer pivot is `Send.bodies` (recipient_x25519_pub → ciphertext) instead of `Send.body`; the inbox grows COUPLES/FAMILIARS sidebar grouping with derived names; the empty-state CTA opens a two-step wizard that matches the mocks at `mocks/v0.3/`. Receivers' invite-preview screen now shows the full member roster including familiars.

**Architecture:** The `Room` model is rebuilt around a `members: List<Member>` list with optional `name`. `RoomKeyCache` becomes a per-(roomId, peerX25519Pub) cache because each member of a room derives N−1 pairwise keys. The send path computes the bodies map by iterating other members. The receive path looks up the addressed key by sender's pubkey. A new `OwnedBots` provider exposes the local user's familiars to the picker. Two new screens (`CreateChatPickScreen`, `CreateChatInviteScreen`) live under `app/lib/screens/create_chat/`. Sidebar/drawer/NavRail render two static section headers (COUPLES, FAMILIARS) populated from the room shape.

**Tech Stack:** Flutter 3.x, Dart 3, Riverpod 2.x, `web_socket_channel`, `cryptography` (existing), `flutter_secure_storage` (existing). No new dependencies.

**Out of scope:** Server changes (separate plan `2026-06-10-v0.3-server.md`). Bot binary (separate plan `2026-06-10-v0.3-bot.md`). Native mobile packaging beyond what v0.2 ships. Forward secrecy on departure. Multi-pair group rooms. Read receipts.

**Contract dependency on the server plan:**
- Server's `Rooms` frame in v0.3 must include `owned_bots: List<Member>` so the picker can list the user's familiars even when they aren't yet in any room. **This is a spec amendment proposed in the server plan's Self-Review §SR-5.** If Court prefers a separate `GET /accounts/owned-bots` REST surface, swap the data source — the client's `ownedBotsProvider` is the single point of change.

**Spec sections implemented:** §3.1 (room shapes), §3.2 (monogamy invariant — UX), §5.1 (CreateRoom + invite preview), §5.2 (Joining), §6.2 (pairwise fan-out encrypt), §6.4 (replay decrypt), §7.1 (room naming derivation), §7.2 (rename UI), §7.3 (sidebar grouping), §7.4 (LeaveRoom UI), §8.2 (wire-frame mirrors), §10.2 (client migration coordination).

## Conventions & Test Infrastructure

- **Tests use `flutter_test` widget tests** for screens and `dart:test` for pure logic.
- **No real WSS**: tests inject a `LiveConnection` fake (existing pattern from v0.2 — `connectForTest`).
- **Riverpod scopes:** wrap each widget test in `ProviderScope(overrides: ...)` with the relevant providers seeded.
- **Wire format invariants:** `SendFrame.bodies` is a `Map<String, String>` (recipient_x25519_pub_b64 → ciphertext); `MessageFrame.body` is a single string (the addressed ciphertext to this recipient).
- **Mobile-first:** every new screen must respond to phone-narrow widths (mocks already do at <600px). Use `LayoutBuilder`.
- **No emoji unless mocks have them.** The mocks do have sage / mauve / wine accent dots and the leaf icon; no other emoji.
- **No backwards-compat shims.** v0.2 wire types are deleted. Court+Kaitlyn install v0.3 binaries before server deploys.
- **Commit cadence:** one commit per task. Required checks: `dart format --output=none --set-exit-if-changed .`, `flutter analyze`, `flutter test`.

```dart
import 'package:flutter/foundation.dart';
import '../wire/frames.dart';

@immutable
class Room {
  const Room({
    required this.roomId,
    required this.name,
    required this.members,
    required this.createdAt,
  });

  final String roomId;
  final String name;
  final List<Member> members;
  final DateTime createdAt;

  /// Spec §7.1 derived display name. Returns `name` verbatim when non-empty.
  String displayName(String selfUsername) {
    if (name.isNotEmpty) return name;
    final others = members.where((m) => m.username != selfUsername).toList();
    final humans = others.where((m) => !m.isBot).map(_capitalize).toList()..sort();
    final bots = others.where((m) => m.isBot).map(_botLabel).toList()..sort();
    return [...humans, ...bots].join(' + ');
  }

  RoomShape shape(String selfUsername) {
    final others = members.where((m) => m.username != selfUsername).toList();
    final humans = others.where((m) => !m.isBot).length;
    final bots = others.where((m) => m.isBot).length;
    if (humans == 1 && bots == 0) return RoomShape.couplesOnly;
    return RoomShape.familiars;
  }

  Member? memberByPubkey(String x25519PubBase64) {
    for (final m in members) {
      if (m.x25519PubBase64 == x25519PubBase64) return m;
    }
    return null;
  }

  Member? memberByUsername(String username) {
    for (final m in members) {
      if (m.username == username) return m;
    }
    return null;
  }
}

enum RoomShape { couplesOnly, familiars }

String _capitalize(Member m) => m.username.isEmpty ? '' :
    m.username[0].toUpperCase() + m.username.substring(1);

/// Strip "<owner>-" prefix from a familiar's username and Title-Case the rest.
/// Bot username convention is `<owner>-<label>` (see server plan Task 17).
String _botLabel(Member b) {
  final dash = b.username.indexOf('-');
  final label = dash >= 0 ? b.username.substring(dash + 1) : b.username;
  if (label.isEmpty) return label;
  return label[0].toUpperCase() + label.substring(1);
}
```

```dart
  switch (f) {
    case RoomsFrame(:final rooms, :final ownedBots):
      final mapped = rooms.map((r) => Room(
        roomId: r.roomId, name: r.name, members: r.members, createdAt: r.createdAt,
      )).toList(growable: false);
      ref.read(inboxStateProvider.notifier).setRooms(mapped);
      ref.read(ownedBotsProvider.notifier).set(ownedBots);
      for (final r in mapped) { _subscribe(r.roomId); }

    case RoomCreatedFrame(:final roomId, :final name, :final members, :final pendingInvite):
      final cur = ref.read(inboxStateProvider).rooms.toList();
      cur.removeWhere((r) => r.roomId == roomId);
      cur.add(Room(roomId: roomId, name: name, members: members, createdAt: DateTime.now().toUtc()));
      ref.read(inboxStateProvider.notifier).setRooms(cur);
      _subscribe(roomId);
      // pendingInvite is consumed by the CreateChatInviteScreen via a
      // dedicated `pendingInviteFor(roomId)` provider keyed off the same
      // frame — surface it through an outbox-style provider:
      if (pendingInvite != null) {
        ref.read(pendingInvitesProvider.notifier).set(roomId, pendingInvite);
      }

    case InviteConsumedFrame(:final roomId, :final name, :final members):
      final cur = ref.read(inboxStateProvider).rooms.toList();
      cur.removeWhere((r) => r.roomId == roomId);
      cur.add(Room(roomId: roomId, name: name, members: members, createdAt: DateTime.now().toUtc()));
      ref.read(inboxStateProvider.notifier).setRooms(cur);
      _subscribe(roomId);

    case RoomRenamedFrame(:final roomId, :final name):
      ref.read(inboxStateProvider.notifier).renameRoom(roomId, name);

    case MemberLeftFrame(:final roomId, :final username):
      ref.read(inboxStateProvider.notifier).removeMember(roomId, username);

    case MessageFrame():
      await _ingestMessage(f);

    case InviteCreatedFrame():
    case RoomErrorFrame():
      // routed to LivePairingTransport / CreateChatInviteScreen.
      break;
  }
}

Future<void> _ingestMessage(MessageFrame f) async {
  final inbox = ref.read(inboxStateProvider);
  final room = inbox.rooms.firstWhereOrNull((r) => r.roomId == f.roomId);
  if (room == null) return;
  final me = await ref.read(currentIdentityProvider.future);
  final sender = room.memberByUsername(f.from);
  if (sender == null) return;
  final senderPub = sender.x25519PubBase64;
  final key = await ref.read(roomKeyCacheProvider).getOrDeriveFor(
        roomId: room.roomId, peerX25519PubBase64: senderPub, me: me,
      );
  final plaintext = await decryptIncoming(key, f.body);
  ref.read(messageStoreProvider(f.roomId).notifier).add(
    Msg(id: f.id, from: f.from, to: f.roomId, body: plaintext, ts: f.ts, replayed: f.replayed),
  );
}
```

```dart
import 'dart:convert';
import 'dart:typed_data';

import 'package:flutter_riverpod/flutter_riverpod.dart';

import '../crypto/ecdh.dart';
import '../identity/keypair.dart';

class RoomKeyCache {
  final Map<String, Uint8List> _keys = {};

  String _k(String roomId, String peerB64) => '$roomId|$peerB64';

  Future<Uint8List> getOrDeriveFor({
    required String roomId,
    required String peerX25519PubBase64,
    required DerivedIdentity me,
  }) async {
    final key = _k(roomId, peerX25519PubBase64);
    final cached = _keys[key];
    if (cached != null) return cached;
    final peerPub = Uint8List.fromList(base64.decode(peerX25519PubBase64));
    final derived = await deriveRoomKey(me: me, peerX25519Pub: peerPub, roomId: roomId);
    _keys[key] = derived;
    return derived;
  }

  void invalidate(String roomId) {
    _keys.removeWhere((k, _) => k.startsWith('$roomId|'));
  }

  void clear() => _keys.clear();
}

final roomKeyCacheProvider = Provider<RoomKeyCache>((_) => RoomKeyCache());
```
