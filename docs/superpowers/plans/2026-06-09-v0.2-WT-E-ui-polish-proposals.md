# v0.2 · WT-E — UI polish proposals

Companion to the HTML mocks in `mocks/v0.2/`. Scope: the new auth flow
(`auth_gate`, `signup`, `recovery_confirm`, `signin`) and the new inbox shell
(`sidebar`, `navigation_rail`, `drawer`, `conversation_list_item`,
`conversation_page`). Identity / pairing / crypto / server are out of scope.

---

## 1. What's bland today

The current UI is Twilight-painted Material 3 — palette correct, intent
absent. Auth screens are stock `AppBar` + `FilledButton` over white-ish
canvas; nothing about them says "this is the only chat for two." The sidebar
reads as a settings menu, not a household. Section headers (`COUPLES`,
`FAMILIARS`) are present but float without anchor or visual hierarchy, and the
empty state ("No conversations yet…") sits in the middle of an empty rectangle
like a 404. We have the right ingredients but no point of view.

## 2. Brand-grounded design direction

The Twilight palette already names its metaphor: *"the hour after the lamps
come on — aubergine canvas, pale rose, sage in the corner of the eye."* The
positioning doc reinforces it: *"Speak to one couple, not a market."*
*"Quietly proud, not preachy."* *"Anchor every privacy claim in a structural
fact."* That gives us our directive: **editorial restraint** — typography
that reads like a printed page rather than a SaaS dashboard, a single sage
hairline as the recurring signature mark (the "sage in the corner of the
eye"), and copy that addresses two specific people. No animation theater.
No glassmorphism. No blob shapes. The lamp is on; we're inside.

## 3. Five concrete upgrades, ordered by ROI

### Upgrade 1 — Editorial type scale and a display font ramp

**Why it matters.** Inter at default weights makes every screen feel like
the same screen. A single typeface used at multiple weights and tracking
values gives the same calm-but-deliberate voice we want from the copy. Inter
is already loaded; we just need to use it like a designer would.

**Mock reference.** `mocks/v0.2/welcome.html` headline (Inter Tight at 44/-0.025em),
`signup-phrase.html` numbered word grid (mono numerals, display-weight words).

**Code patch.** Add a typography helper in `app/lib/theme/twilight.dart`
(under the existing `buildTwilightTheme`). No new dependency — Flutter ships
`fontVariations` and `letterSpacing` out of the box.

```dart
// app/lib/theme/twilight.dart — add below buildTwilightTheme()

/// Editorial type scale. One typeface (Inter), multiple voices.
/// Use these instead of raw Text(style: ...) so every screen rhymes.
class TwilightType {
  static const _f = 'Inter';

  /// Display headline — auth screens, empty states. Tight, weighty.
  static const display = TextStyle(
    fontFamily: _f,
    fontSize: 32,
    height: 1.12,
    fontWeight: FontWeight.w500,
    letterSpacing: -0.7,
    color: TwilightColors.textPrimary,
  );

  /// Mid headline — section titles within a screen.
  static const title = TextStyle(
    fontFamily: _f,
    fontSize: 20,
    height: 1.2,
    fontWeight: FontWeight.w500,
    letterSpacing: -0.25,
    color: TwilightColors.textPrimary,
  );

  /// Body — composer text, bubble text, prose.
  static const body = TextStyle(
    fontFamily: _f,
    fontSize: 15,
    height: 1.5,
    fontWeight: FontWeight.w400,
    letterSpacing: -0.05,
    color: TwilightColors.textPrimary,
  );

  /// Annotation — sidebar section headers, step markers, kbd hints.
  /// Wide tracking so it reads like a footer caption.
  static const annotation = TextStyle(
    fontFamily: _f,
    fontSize: 11,
    height: 1.0,
    fontWeight: FontWeight.w500,
    letterSpacing: 2.4, // 0.22em at 11px
    color: TwilightColors.textMuted,
  );

  /// Muted body — sub-copy under headlines.
  static const lede = TextStyle(
    fontFamily: _f,
    fontSize: 14,
    height: 1.6,
    fontWeight: FontWeight.w400,
    letterSpacing: -0.05,
    color: TwilightColors.textMuted,
  );
}
```

### Upgrade 2 — The sage hairline as a recurring signature mark

**Why it matters.** Right now `accentFamiliar` (sage) is unused except inside
the `EmojiPicker` indicator. The palette includes a `ruleStrong` token *named
for this purpose* — bring it forward. Use it once in each chrome surface (the
right edge of the sidebar, the selected-row accent, the auth step indicator),
never anywhere else. One device, repeated → identity.

**Mock reference.** Visible in `inbox-with-rooms.html` (right edge of sidebar,
selected row accent at left), `signup-phrase.html` (step marker bar), all
`signup-*` "Step n" preludes.

**Code patch.** Replace the existing `Container(height: 1, color: borderSoft)`
in `sidebar.dart` and replace the sidebar wrapper with one that draws the
hairline. The whole change is contained to `sidebar.dart` and
`conversation_list_item.dart`.

class _SageHairline extends StatelessWidget {
  const _SageHairline();
  @override
  Widget build(BuildContext context) {
    return const DecoratedBox(
      decoration: BoxDecoration(
        gradient: LinearGradient(
          begin: Alignment.topCenter,
          end: Alignment.bottomCenter,
          colors: [
            Color(0x00000000),                     // transparent
            Color(0x594F7A5E),                     // ruleStrong @ 35%
            Color(0x594F7A5E),
            Color(0x00000000),
          ],
          stops: [0.0, 0.2, 0.8, 1.0],

      ),
    );
  }
}

### Upgrade 3 — The 12-word phrase as a printed page

**Why it matters.** Today the recovery phrase renders as a `GridView` with
`childAspectRatio: 3`. It looks like a CSV. This screen is the moment the
user is told *"this is the only way back"* — it deserves to look like
something they'd want to write down. Numbered cells, mono ordinals, dotted
baselines, sage-marked 3/7/11 to telegraph the upcoming confirm step.

**Mock reference.** `mocks/v0.2/signup-phrase.html`.

**Code patch.** Replace the `GridView.builder` block in
`app/lib/screens/auth/signup.dart`'s `_step2`:

of arrival ("the hour after the lamps come on"). Cost: one
`AnimationController` in `InboxShell`, careful guarding so widget tests
don't time out waiting on the animation. Worth doing once the inbox feels
right — premature today.

**A second variant of the recovery-phrase card for "I read it aloud" couples.**
For partners who'll read words to each other over the phone, an optional
"read mode" that enlarges the words to display size, dims everything else,
and hides the numbers behind a tap. Different gesture, same data. Stretch
because it doubles the surface area of the screen and we haven't observed
the use yet.

**Avatar with two-tone monogram for the user's own footer handle.**
`@court` in the sidebar footer renders today as plain text. A small
two-color disc (aubergine fill, rose initial) would make the footer feel
like a signature stamp. Cost: a `CircleAvatar` + 24×24 draw. Defer because
the user's own face appears nowhere else in v0.2; introducing it only in
the footer creates an asymmetry that wants explaining.

## 5. Recommendation

If I were the engineer with one PR's worth of time:

1. **Ship Upgrade 1 (type scale + auth screens redesign)** — biggest visual
   delta per line of code, touches three of the four "first-impression"
   screens, no test risk.
2. **Ship Upgrade 4 (empty-state as invitation)** — fixes the saddest
   surface in the app (signed in, nothing to look at) and is fully
   self-contained in `inbox_shell.dart`.
3. **Ship Upgrade 5 (encryption seal in topbar)** — small wire-up,
   anchors the brand claim every time the user opens a chat. Reuses
   `accentFamiliar` correctly for the first time.

**Defer Upgrade 2 (sage hairline)** to a second pass. It's the most
*designed* of the five but it interacts with the sidebar layout and the
selected-row visual; better to do it deliberately after the typography
upgrade lands so we can judge whether the hairline is the right "signature
mark" or whether the new type alone already carries it.

**Defer Upgrade 3 (phrase as printed page)** unless time allows — it's
high-leverage but the existing implementation works and the recovery flow
is rarely seen. It's the one to revisit when we polish the marketing site,
because a screenshot of that page belongs on the marketing site.
